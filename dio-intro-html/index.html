Introdução ao HTML

HTML é a linguagem usada pra criar a estrutura de páginas na web. Ele não é uma linguagem de programação (tipo JavaScript), mas sim linguagem de marcação, que serve pra marcar os elementos da página: título, parágrafo, imagem, link, lista e por aí vai.

Tudo em HTML é feito com tags, que são como blocos que dizem pro navegador o que cada parte da página representa. Essas tags são escritas entre sinais de menor e maior (tipo <p> ou <img>), e muitas delas têm uma tag de abertura e uma de fechamento (tipo <h1>...</h1>).

A estrutura básica de uma página HTML começa com:

<!DOCTYPE html> – diz que o documento é HTML5.
<html> – envolve todo o conteúdo.
<head> – parte que vai coisas invisíveis pro usuário, tipo título da aba, link com arquivos CSS, etc.
<body> – onde fica o conteúdo visível: textos, imagens, links, botões...

Exemplos de tags importantes:

<h1> até <h6> – títulos, do maior pro menor.
<p> – parágrafos.
<a href="..."> – links.
<img src="..." alt="..."> – imagens.
<ul> e <li> – listas não ordenadas.
<div> e <span> – usadas pra organizar e agrupar elementos.

O HTML é o alicerce de qualquer site. Ele sozinho já monta a estrutura, mas precisa de CSS pra estilizar e de JS pra deixar tudo mais interativo.

HTML Semântico

É quando você usa tags que têm significado próprio, em vez de só usar <div> e <span> pra tudo. Isso ajuda o navegador, os mecanismos de busca (como o Google) e até leitores de tela (usados por pessoas com deficiência visual) a entenderem o que cada parte do seu site representa.

Exemplos:

Não Semântico

<div id="menu">...</div>
<div id="conteudo">...</div>
<div id="rodape">...</div>

Semântico

<nav>...</nav>
<main>...</main>
<footer>...</footer>

Tags semânticas mais comuns:

<header> – Cabeçalho (geral ou de uma seção).
<nav> – Área de navegação.
<main> – Conteúdo principal.
<section> – Bloco de conteúdo com tema.
<article> – Conteúdo independente (ex: post, notícia).
<aside> – Conteúdo lateral ou complementar.
<footer> – Rodapé.

Por que usar HTML semântico?

  • Melhora a acessibilidade (para leitores de tela).
  • Facilita o SEO (Google entende melhor seu conteúdo).
  • Deixa o código mais organizado e legível.
  • Ajuda outras pessoas (ou você no futuro) a entenderem o que está acontecendo.

Trabalhando com Formulários

É o jeito de coletar dados do usuário (nome, email, senha...) e enviar pra algum lugar como por exemplo um servidor. Tudo começa com a tag <form>.

Estrutura Básica

<form action=" " method=" ">
</form>
  • action: define pra onde os dados vão (ex: URL do servidor).
  • method: pode ser "get" (envia na URL) ou "post" (envia “escondido”).

Principais Tags usadas dentro de um formulário:

  1. input - Campo genérico que muda conforme o type. (Ex.: <input type="...">)

    2. type="text" – campo de texto.

    type="email" – valida email.

    type="password" – oculta os caracteres.

    type="checkbox" – caixinha de seleção.

    type="radio" – botão de opção única (geralmente em grupo).

    type="number" – só aceita números.

    type="file" – para envio de arquivos.

  2. label - Texto que identifica um campo. Associado com o atributo for.

    3. <label for="nome">Nome:</label>
    <input type="text" name="nome">

  3. textarea - Área de texto maior (tipo “mensagem” ou “comentário”).

    4. <textarea name="mensagem"></textarea>

  4. select e option - Menu suspenso (drop-down).

    5. <select name="cor">
    <option value="azul">Azul</option>
    <option value="vermelho">Vermelho</option>
    </select>

  5. button - Botão que pode ser submit e reset

    6. A tag <button> tem diferentes comportamentos dependendo do valor do atributo type, e isso é útil pra controlar o que o botão vai fazer dentro de um formulário.

    <button type="submit">Enviar</button>

    <button type="reset">Apagar</button>
    • submit: envia o formulário.
    • reset: limpa os campos.

Trabalhando com Mídias

Usamos o HTML pra exibir conteúdos visuais e sonoros na página: imagens, vídeos, áudios e até conteúdos de outras plataformas (como YouTube, mapas, etc.). Usando as tags <img>, <audio>, <video> e <iframe>

  1. Imagens – <img>

    2. Serve pra mostrar imagens na página.


    <img src="caminho-da-imagem.png" alt="Descrição da imagem">
    • src: caminho do arquivo.
    • alt: texto alternativo (acessibilidade).
    • width e height: pra controlar o tamanho.
  2. Áudio – <audio>

    3. Pra tocar sons ou músicas.


    <audio controls>
    <source src="caminho-d-audio.mp3" type="audio/mpeg">
    Seu navegador não suporta áudio.
    </audio>
    • controls: mostra os botões de play/pause.
    • autoplay: reproduz a mídia automaticamente.
    • Conseguimos usar outros formatos como .mp3, .ogg, .wav.
  3. Vídeo - <video>

    4. Pra incorporar vídeos direto na página.


    <video controls width="300">
    <source src="caminho-do-video.mp4" type="video/mp4">
    Seu navegador não suporta vídeo.
    </video>
    • controls: mostra os botões de play/pause.
    • autoplay: reproduz a mídia automaticamente.
    • loop: repete a mídia.
    • muted: deixa a mídia no mudo.
    • width: para controlar o tamanho.
  4. Track - <track>

    5. A tag <track> é útil quando você quer adicionar legendas, descrições ou traduções a um vídeo ou áudio no HTML. Ela funciona junto com as tags <video> ou <audio>, nunca sozinha.

    <video controls width="300">
    <source src="caminho-do-video.mp4" type="video/mp4">
    <track src="legenda.vtt" kind="subtitles" srclang="pt" label="Português" default>
    Seu navegador não suporta vídeo.
    </video>
